## Getting Started

### Prerequisites

- Node.js 20+
- npm

### Install

```bash
git clone https://github.com/kimmania/game-slot-heist.git
cd game-slot-heist
npm install
```

### Development

```bash
npm run dev
```

Vite dev server starts on `http://localhost:5173` (or next available port).

### Production Build

```bash
npm run build
```

Output goes to `dist/`. The build includes:
- JS/CSS bundles with hashed filenames
- `manifest.webmanifest` (auto-generated by `vite-plugin-pwa`)
- Service worker (`sw.js`) + Workbox runtime
- Precached assets for offline play

### Preview

```bash
npm run preview
```

Serves the `dist/` build locally on `http://localhost:4173`.

## Configuration

### Vite Base Path

`vite.config.ts` sets `base: '/game-slot-heist/'` вЂ” this must match the GitHub Pages repository name so asset links resolve correctly.

### PWA Manifest

Generated automatically from `vite.config.ts` в†’ `manifest` options:
- `display: standalone`
- `orientation: portrait`
- Theme/background color: `#0f1115`
- Icons referenced from `public/icons/`

### GitHub Actions Deploy

On every push to `main`, the workflow:
1. Checks out the repo
2. Installs dependencies (`npm ci`)
3. Builds the project (`npm run build`)
4. Uploads `dist/` as a Pages artifact
5. Deploys to GitHub Pages

See `.github/workflows/deploy.yml`.

## Save State

Game state is persisted to `localStorage` under key `slot-heist-save`:

- `bank` вЂ” current bankroll
- `betIdx` вЂ” selected bet index
- `level` / `xp` вЂ” progression
- `dailyLogin` вЂ” last daily bonus timestamp
- `recentWins` / `topWins` вЂ” leaderboard data
- `turbo` вЂ” user preference
- `sound` вЂ” audio enabled (default `true`)
- `hasSeenHelp` вЂ” first-time help flag

**Reset** wipes all of the above and restores defaults.
